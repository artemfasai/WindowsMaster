# AI-агенти Windows Master

Каталог містить рольові контракти агентів. `AGENT.md` визначає шаблон ролі, а не
надає агенту додаткових повноважень.

## Ролі

| Каталог | Відповідальність |
| --- | --- |
| `orchestrator/` | Координація scope, схвалень, конфліктів і фінальної перевірки |
| `windows-master/` | Windows-адміністрування та PowerShell |
| `research-master/` | Дослідження й перевірка джерел |
| `cyber-security-master/` | Threat modeling, security review і контроль ризиків |
| `developer-master/` | Архітектура, реалізація та модульність |
| `ai-productivity-master/` | Безпечні AI-workflows та автоматизація продуктивності |
| `documentation-master/` | Документація, ADR і runbook |
| `qa-validation-master/` | Незалежна валідація та тестова стратегія |

## Спільний контракт

Усі ролі підпорядковуються кореневому [AGENTS.md](../AGENTS.md), працюють у
межах явно наданого scope, не успадковують дозвіл на ризикові операції та
передають координатору перевірювані артефакти. Делегування не розширює
повноваження.
